The EUW app extends standard Frappe DocTypes with custom fields to support specific business requirements. Custom fields are defined in the fixtures system and automatically applied during installation.

## Overview

Custom fields in EUW are used to:
- Add business-specific data to standard DocTypes
- Enhance functionality of existing forms
- Support custom workflows and integrations
- Maintain data consistency across the application

## Custom Field Structure

Each custom field is defined with the following key properties:

```json
{
  "dt": "DocType name",
  "fieldname": "unique_field_name",
  "fieldtype": "Field type",
  "label": "User-visible label",
  "insert_after": "existing_field",
  "options": "Link target or options"
}
```

## Field Properties

### Essential Properties

- **dt**: The DocType being customized
- **fieldname**: Unique identifier (must start with `custom_` for custom fields)
- **fieldtype**: Data type (Link, Data, Text, Date, etc.)
- **label**: Display name shown to users
- **insert_after**: Position in the form layout

### Optional Properties

- **options**: For Link fields (target DocType) or Select fields (choices)
- **depends_on**: JavaScript expression to show/hide field
- **mandatory_depends_on**: Conditional requirement
- **fetch_from**: Auto-populate from linked DocType
- **read_only**: Make field non-editable
- **hidden**: Hide field from view
- **translatable**: Allow field translation

## Best Practices

<Note>
**Naming Convention**: Always prefix custom field names with `custom_` to distinguish them from standard fields and avoid conflicts during upgrades.
</Note>

<Note>
**Field Positioning**: Use `insert_after` to place fields logically within the form. Group related fields using Section Breaks.
</Note>

<Note>
**Link Fields**: When using Link fields, ensure the target DocType exists and has appropriate permissions configured.
</Note>

## Creating Custom Fields

### Using Bench Export

To export custom fields for your app:

```bash
bench --site [site-name] export-fixtures
```

This exports all fixtures defined in `hooks.py` including custom fields.

### Manual Definition

You can also manually define custom fields in `fixtures/custom_field.json`:

```json
[
  {
    "dt": "Your DocType",
    "fieldname": "custom_your_field",
    "fieldtype": "Data",
    "label": "Your Field Label",
    "insert_after": "existing_field"
  }
]
```

## Importing Custom Fields

Custom fields are automatically imported when:

1. **Installing the app**:
   ```bash
   bench --site [site-name] install-app euw
   ```

2. **Migrating the site**:
   ```bash
   bench --site [site-name] migrate
   ```

3. **Manually importing fixtures**:
   ```bash
   bench --site [site-name] import-doc fixtures/custom_field.json
   ```

## Conditional Fields

Use `depends_on` to show fields conditionally:

```json /home/daytona/workspace/source/euw/fixtures/custom_field.json:1354-1400
{
  "dt": "Employee",
  "fieldname": "health_insurance_no",
  "fieldtype": "Data",
  "label": "Health Insurance No",
  "depends_on": "eval:doc.health_insurance_provider"
}
```

This field only appears when `health_insurance_provider` has a value.

## Troubleshooting

<Accordion title="Custom field not appearing">
1. Check that the field is in `fixtures/custom_field.json`
2. Verify fixtures are listed in `hooks.py`
3. Run `bench migrate` to apply changes
4. Clear cache with `bench clear-cache`
</Accordion>

<Accordion title="Field shows in wrong position">
Check the `insert_after` value points to an existing field in the DocType.
</Accordion>

<Accordion title="Link field not working">
Ensure:
- The `options` value matches the target DocType exactly
- The target DocType exists in the system
- User has read permissions on the target DocType
</Accordion>
