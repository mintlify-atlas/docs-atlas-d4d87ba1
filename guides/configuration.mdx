The EUW app's behavior is controlled through configuration files that define hooks, fixtures, and desktop settings. Understanding these configurations is essential for customizing and extending the app.

## Configuration Files Overview

EUW uses three primary configuration files:

```
euw/
├── hooks.py          # Main app configuration and hooks
├── config/
│   └── desktop.py    # Desktop module configuration
└── fixtures/         # UI customization data
```

## hooks.py - Main Configuration

The `hooks.py` file is the heart of the app's configuration.

### App Metadata

```python /home/daytona/workspace/source/euw/hooks.py:1-8
app_name = "euw"
app_title = "EUW"
app_publisher = "Aseel Gh"
app_description = "EUW app for Workorder"
app_email = "aseel.gharbia@gmail.com"
app_license = "MIT"
```

**Key Metadata Fields:**
- `app_name`: Internal identifier (used in installation)
- `app_title`: Display name in the UI
- `app_publisher`: Developer/organization name
- `app_description`: Brief description of functionality
- `app_license`: Software license type

### Fixtures Configuration


This defines which customizations are exported/imported as fixtures. See the [Fixtures Guide](/guides/fixtures) for details.

### Document Events

Hook into document lifecycle events:

```python /home/daytona/workspace/source/euw/hooks.py:117-127
# doc_events = {
#   "*": {
#       "on_update": "method",
#       "on_cancel": "method",
#       "on_trash": "method"
#   }
# }
```

<Accordion title="Document Event Example">
```python
doc_events = {
    "Sales Invoice": {
        "on_submit": "euw.events.sales_invoice.on_submit",
        "validate": "euw.events.sales_invoice.validate"
    },
    "*": {
        "before_insert": "euw.events.common.set_defaults"
    }
}
```

Then create the event handlers:

```python
# euw/events/sales_invoice.py
def on_submit(doc, method):
    # Logic when sales invoice is submitted
    pass

def validate(doc, method):
    # Validation logic
    pass
```
</Accordion>

## Environment Variables

Configure app behavior through environment variables:

### In site_config.json

```json

  "euw_settings": {
    "enable_feature_x": true,
    "api_endpoint": "https://api.example.com"
  }
}
```

Access in code:

```python
import frappe

settings = frappe.conf.get("euw_settings", {})
if settings.get("enable_feature_x"):
    # Feature logic
    pass
```

## Custom Settings DocType

For user-configurable settings, create a Settings DocType:

```python
# Create EUW Settings DocType with fields:
# - enable_auto_approval (Check)
# - default_warehouse (Link to Warehouse)
# - notification_email (Data)

# Access settings:
import frappe

def get_euw_settings():
    return frappe.get_single("EUW Settings")

settings = get_euw_settings()
if settings.enable_auto_approval:
    # Auto-approve logic
    pass
```

## Best Practices

<Note>
**Organize by Feature**: Group related hooks and configurations together with comments.
</Note>

<Note>
**Use Hooks Over Core Modifications**: Always use hooks to customize behavior rather than modifying core Frappe/ERPNext files.
</Note>

<Note>
**Document Custom Hooks**: Add docstrings and comments explaining the purpose of each hook.
</Note>

<Note>
**Test Hook Functions**: Write unit tests for custom hook functions to ensure they work correctly.
</Note>

## Common Configuration Patterns

### Auto-set Field Values

```python
# hooks.py
doc_events = {
    "Item": {
        "before_save": "euw.utils.item.set_defaults"
    }
}

# euw/utils/item.py
def set_defaults(doc, method):
    if not doc.custom_applicability:
        doc.custom_applicability = "General"
```

### Validate Documents

```python
# hooks.py
doc_events = {
    "Sales Order": {
        "validate": "euw.validations.sales_order.validate"
    }
}

# euw/validations/sales_order.py
import frappe

def validate(doc, method):
    if doc.custom_field_x and not doc.custom_field_y:
        frappe.throw("Custom Field Y is required when Custom Field X is set")
```

### Send Notifications

```python
# hooks.py
doc_events = {
    "Work Order": {
        "on_submit": "euw.notifications.work_order.notify_team"
    }
}

# euw/notifications/work_order.py
import frappe

def notify_team(doc, method):
    frappe.sendmail(
        recipients=["team@example.com"],
        subject=f"Work Order {doc.name} Submitted",
        message=f"Work Order {doc.name} has been submitted and is ready for processing."
    )
```

## Troubleshooting

<Accordion title="Hooks not executing">
1. Clear cache: `bench clear-cache`
2. Restart bench: `bench restart`
3. Check for Python syntax errors in hook files
4. Verify function paths are correct
</Accordion>

<Accordion title="Module not showing in Desk">
1. Check `config/desktop.py` syntax
2. Rebuild desk: `bench build`
3. Clear cache and reload
</Accordion>

<Accordion title="Import errors in hooks">
1. Ensure all referenced modules exist
2. Check Python path and imports
3. Verify function names are correct
</Accordion>
