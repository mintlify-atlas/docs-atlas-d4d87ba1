## Overview

EUW is a Frappe framework application designed to extend ERPNext with specialized functionality for work order management. The app integrates seamlessly with the Frappe/ERPNext ecosystem while maintaining a modular, maintainable architecture.

## Role in the Frappe Ecosystem

EUW operates as a custom Frappe app that:

- **Extends ERPNext**: Built on top of ERPNext's core functionality, inheriting its document management, workflow, and permission systems
- **Adds Domain-Specific DocTypes**: Introduces specialized document types for work order scenarios (DDS, NRC, Applicability)
- **Customizes Standard DocTypes**: Enhances existing ERPNext documents like Item, Asset, Employee, and Company with additional fields
- **Maintains Compatibility**: Uses Frappe's hooks system to integrate without modifying core files

<Note>
EUW follows Frappe best practices by using fixtures for customizations, ensuring all changes are version-controlled and reproducible across environments.
</Note>

## Module Structure

The EUW application follows Frappe's standard directory structure:

```
euw/
├── euw/                      # Main Python module
│   ├── doctype/             # Custom DocTypes
│   │   ├── applicability/   # Applicability DocType
│   │   ├── dds/            # DDS (Deferred Defect Sheet) DocType
│   │   └── nrc/            # NRC (Non-Routine Card) DocType
│   └── custom/             # Custom field definitions
│       ├── asset.json
│       ├── customer.json
│       ├── issue.json
│       ├── item.json
│       └── ...
├── fixtures/               # Fixture data for import/export
│   ├── custom_field.json
│   ├── property_setter.json
│   ├── client_script.json
│   ├── print_format.json
│   └── translation.json
├── config/                 # App configuration
├── templates/             # Jinja templates
└── hooks.py              # Frappe hooks configuration
```

## Integration Points

### Fixtures System

EUW uses Frappe's fixtures system to manage customizations:

```python hooks.py
fixtures = [
    'Client Script',
    'Custom Field',
    'Property Setter',
    'Print Format',
    'Translation'
]
```

This ensures all customizations are:
- Version controlled
- Reproducible across environments
- Exportable and importable
- Tracked in the codebase

### Standard DocType Extensions

EUW extends multiple ERPNext DocTypes with custom fields:

- **Item**: Adds applicability linking
- **Asset**: Adds branch field
- **Employee**: Adds HR-specific fields (employment type, grade, approvers)
- **Company**: Adds HR & Payroll settings
- **Department**: Adds approver workflows
- **Project/Task**: Adds expense tracking fields

## Design Principles

### Non-Invasive Customization

EUW follows Frappe best practices:
- Never modifies core files
- Uses Custom Fields instead of patching DocTypes
- Leverages hooks for extensions
- Maintains upgrade compatibility

### Separation of Concerns

- **DocTypes**: Core business logic in `euw/euw/doctype/`
- **Customizations**: Field additions in `euw/euw/custom/` and `fixtures/`
- **Configuration**: App settings in `hooks.py`
- **Templates**: UI customizations in `templates/`

### Data-Driven Configuration

Customizations are defined as JSON fixtures, making them:
- Easy to review in version control
- Portable across instances
- Auditable and traceable

## Deployment Architecture

EUW deploys as a standard Frappe app:

1. **Installation**: `bench get-app` to download
2. **Site Installation**: `bench --site [sitename] install-app euw`
3. **Migration**: Frappe automatically creates DocTypes and applies fixtures
4. **Updates**: Standard bench update process

<Warning>
Always test customizations in a development environment before deploying to production.
</Warning>
